Richardson Maturity Model

The Richardson Maturity Model is a way to evaluate RESTful APIs based on their adherence to REST principles. Each level adds more RESTful features.

Level 0: The Swamp of POX (One URI, one HTTP method) - Deprecated

Uses HTTP as a transport for remote interactions, with a single endpoint, like XML-RPC or SOAP services.

Example:
POST /api
Body: { "action": "createUser", "name": "Alice" }

Level 1: Resources (Many URIs, one HTTP method) - Deprecated

Resources get their own URIs (/users, /orders, etc.).

Example:
POST /users  
Body: { "name": "Alice" }

Level 2: HTTP Verbs (Many URIs, multiple HTTP methods)

Uses HTTP methods as verbs (GET, POST, PUT, DELETE) and status codes (200 OK, 404 Not Found, etc.).

Example:
GET /users/123        # Retrieve user with ID 123  
POST /users           # Create new user
PUT /users/123        # Update user  
DELETE /users/123     # Delete user

Level 3: Hypermedia Controls (HATEOAS) - Rare but Powerful

Responses include links to available actions. Clients can discover functionality dynamically (what they can do next).

Example:
Response for GET /users/123:
{
  "id": 123,
  "name": "Alice",
  "links": [
    { "rel": "self", "href": "/users/123" },
    { "rel": "update", "href": "/users/123", "method": "PUT" },
    { "rel": "delete", "href": "/users/123", "method": "DELETE" },
    { "rel": "orders", "href": "/users/123/orders", "method": "GET" }	
  ]
}
