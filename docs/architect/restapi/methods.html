HTTP Methods (HTTP Verbs)

Overview

According to RFC 9110, HTTP defines request methods to interact with a given resource. A request method can be safe, idempotent, or cacheable.
Safe means that it does not change any data on the server (the state of the server). Read-only operation.
Idempotent means that the result (the state of the server) remains unchanged if the same request (operation) is executed several times.
Cacheable means that the response can be stored and reused

Example

PATCH /products/12345 HTTP/1.1
Host: api.ecommerce.com
Content-Type: application/json
Authorization: Bearer <access_token>
{"price":"5000"}

Idempotent PATCH example:
We have a product for which we want to update the price.
Non-idempotent PATCH example:
We have a shopping cart to which we want to add products.
