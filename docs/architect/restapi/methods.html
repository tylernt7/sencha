HTTP Methods

Overview

A request method can be safe, idempotent, or cacheable.
Idempotent means that the result (the state of the server) remains unchanged if the same request (operation) is executed several times.

HTTP request methods (HTTP verbs)

Request Description Note
GET /products Retrieve a list of products
GET /products/{id} Retrieve a specific product by its ID
POST /products Create a new product Create a new entity
PUT /products/{id} Update an existing product by its ID Create a new entity if not exist, modify/replace the entity if exists (idempotent)
DELETE /products/{id} Delete an existing product by its ID
