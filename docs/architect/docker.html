Docker

Docker is a containerization environment (container-based virtualization). Docker is a set of tools that enables you to create, deploy, and run applications using containers. Containers can be run on any machine that has Docker installed: you can run your application on your local machine or on a remote server without the risk of a missing software library or different Java version

Difference between Virtual Machines and Dockers

Both Virtual Machines and Dockers provide isolated environments to run applications
  • A VM virtualizes both the operating system kernel and the application layer, and runs on its own guest operating system
    • VirtualBox, Hyper-V, VMware Workstation
  • A Docker virtualizes only the application layer, and runs on top of the host operating system
    • Dockers bundle their own software, binaries/libraries and configuration files
    • Dockers use fewer resources than virtual machines because of single operating system kernel

Docker Architecture

  • Software Components
    • Docker daemon (dockerd): it manages Docker containers and handles container objects
    • Docker client (docker): it interacts with Docker daemons
  • Object Components
    • Docker container: it runs applications
    • Docker image: it stores applications
    • Docker service (swarm): it controls Docker containers
  • Registry Components
    • Docker repository (Docker Hub or Artifactory): it downloads ("pull") images for use or uploads ("push") images that have been built

Dockerfile, Docker Image, and Docker Container

chroot

A chroot (change root) is an operation in the Unix/Linux operating systems that changes the root directory for the running process. A program that is run in a modified environment (chroot jail) is imprisoned in this directory (a smaller system) and cannot access files outside of it.

A Docker container provides more isolation than chroot by using multiple namespace types, such as pid, user, uts, ipc, net and mnt.

To have a functional chroot environment, the kernel virtual file systems and configuration files also have to be mounted/copied from host to chroot (independent Linux system with a separate file system).
