REST API

Terminology

URL (Uniform Resource Locator): Web address

HTTP (Hypertext Transfer Protocol): Stateless protocol for the transmission of information on the application layer (TCP port 80)

HTTPS (Hypertext Transfer Protocol Secure): Secure communication protocol using encryption (TCP port 443, HTTPS = HTTP + SSL/TLS)

REST API (REpresentational State Transfer - Application Programming Interface): Guiding principles or constraints of the RESTful architecture are:

  • Clientâ€“server architecture: Clients communicating with a server via the Internet
  • Stateless protocol: The server must not retain session/context/cookie information from previous requests
  • Caching: Clients can cache responses on the web
  • Uniform interface: Resource identification, resource manipulation, self-descriptive messages, hypermedia as the engine of application state
  • Layered system: Intermediary servers e.g. proxy, load balancer or security layer
  • Code on demand (optional): Downloading and executing code in the form of Java applets or JavaScripts

Request methods

Request Description Note
GET /coffees Retrieve multiple Coffees with a list of IDs
GET /coffees/:id Retrieve a single Coffee with an ID
POST /coffees Create a Coffee Create a new entity
PUT /coffees/:id Update a Coffee Create a new entity if not exist, modify/replace the entity if exists (idempotent: unchanged if done several times)
DELETE /coffees Delete a Coffee
