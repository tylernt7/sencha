Threading Models

Aspect Spring Boot (MVC) Spring WebFlux (Reactive)
Thread Model Multi-threaded Event-loop
Default Server Tomcat Netty
Blocking Behavior Blocking I/O (e.g., JDBC, RestTemplate) Non-blocking I/O (e.g., R2DBC, WebClient)
Execution Model Synchronous request/response processing Asynchronous data streams
Thread Usage One thread per request One thread – many requests
Scalability Limited by thread pool size Higher due to non-blocking I/O

Spring Boot (Synch) + Virtual Threads (Asynch) = Spring WebFlux (Asynch)

  • Virtual Threads (JDK 21) are lightweight threads managed by the JVM, not the OS.
  • Massive concurrency with minimal overhead.
  • Ideal for I/O-bound tasks (DB connectivity, HTTP client).
  • You can enable virtual threads in application.yml (application.properties).
Thread.startVirtualThread(() -> {
    System.out.println("Running in a virtual thread");
});

// Or using an executor:

try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> System.out.println("Hello from virtual thread"));
}
