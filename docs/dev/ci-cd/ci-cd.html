CI/CD pipeline

It automates the analyzing, building, deploying, and testing of software applications

Continuous Integration (CI): folyamatos integráció

Trivy: Vulnerability scanner
SonarQube: Static code analysis
Docker Build: Docker images (Kaniko)
JUnit/Dotnet Test: Unit tests
Newman: Automated tests

Continuous Delivery/Deployment (CD): folyamatos szállítás/telepítés

Helm Deploy: Package manager for Kubernetes

Typical workflow of deployment

1. Clone repository: git clone
2. Update tag: yq (update the image tag field in a values.yaml file with the short SHA of the current commit)
3. Login to the registry: helm registry login
4. Upgrade Helm: helm upgrade (install the Helm charts to the Harbor registry)
