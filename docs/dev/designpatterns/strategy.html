Strategy

Intent

  • Define a family of algorithms, encapsulate each one, and make them interchangeable. Strategy lets the algorithm vary independently from clients that use it.
  • Examples:
    • Any of modes of transportation will get a traveler to the airport, and they can be used interchangeably. The traveler must choose the Strategy based on trade-offs between cost, convenience, and time.
    • There are different ways to pay (PayPal, Credit Card, etc.) when you want to buy a product in an e-commerce. Purchase is our goal/context which provides all the possible strategies that the client can use to pay.
    • The client can switch between different sorting strategies including Quicksort, Shellsort, and Mergesort.

Structure

Structural Code

Context

The "context" is the code (does not need to be a class) where "strategic method(s)" are called and the strategic logic is kept.
