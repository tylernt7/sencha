Terminology

Verification Are you doing what is written?
  • Confirmation by examination and through provision of objective evidence that specified requirements have been fulfilled [ISO 9000]
  • Evaluating a work product, component or system to determine whether it meets the requirements set
  • Checking the fulfillment of a requirement by an implemented system
  • Are we building the product right?
  • Is the system correct to specification?
  • Is the deliverable built according to the specification?
  • Same as testing
Validation Is the description good?
  • Confirmation by examination and through provision of objective evidence that the requirements for a specific intended use or application have been fulfilled [ISO 9000]
  • Evaluating a work product, component or system to determine whether it meets the user/customer needs and requirements
  • Checking whether documented requirements match the stakeholdersâ€™ needs. Discover errors in the requirements: ambiguity, incompleteness, contradictions and incomprehensibility
  • Are we building the right product?
  • Is this the right specification?
  • Is the deliverable fit for purpose, e.g. does it provide a solution to the problem?
Traceability
  • Test conditions should be able to be linked back to their sources in the test basis (e.g. requirements)
  • The ability to trace a requirement (1) back to its origins (stakeholders, documents, rationale, etc.), (2) forward to its implementation in design and code, (3) to requirements it depends on (and vice-versa)
  • Vertical traceability shows relationships between requirements and horizontal traceability displays relationships between requirements and test cases
