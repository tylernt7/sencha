Traceability
  • Test conditions should be able to be linked back to their sources in the test basis (e.g. requirements)
  • The ability to trace a requirement (1) back to its origins (stakeholders, documents, rationale, etc.), (2) forward to its implementation in design and code, (3) to requirements it depends on (and vice-versa)
  • Vertical traceability shows relationships between requirements and horizontal traceability displays relationships between requirements and test cases

Traceability (from Tata, TCS)

A Traceability Matrix is a crucial artifact that identifies the relationship among various components of a software system. It enables any software component to be traced back to the requirement.
Using a Traceability Matrix, you can identify all the impacted items while analyzing the changes to a requirement.
There are different types of Traceability Matrices that are used for impact analysis:
1. Horizontal Traceability Matrix helps to identify if a change in one requirement also impacts other requirements.
2. Vertical Traceability Matrix helps to trace corresponding items amongst the different work products throughout the software lifecycle.
