Hello World

Angular is a free, open-source TypeScript-based single-page application (SPA) framework run on Node.js.

Run the following commands in the terminal:

npm install -g @angular/cli
It installs the latest stable version of the Angular Command Line Interface globally (e.g. for ng new or ng serve).

setx PATH "%PATH%;c:\Users\user\AppData\Roaming\npm"
It adds the npm global directory to PATH for the ng command.

ng new hello-world
It creates a new Angular project.

ng generate component hello
It creates a new component within the Angular project.

Write the following HTML code (template) in src/app/hello/hello.component.html:

<h1>{{ title }}</h1>

Write the following TypeScript code in src/app/hello/hello.component.ts:

import { Component } from '@angular/core';
	
@Component({
  selector: 'app-hello',
  templateUrl: './hello.component.html',
  styleUrl: './hello.component.css'
})
export class HelloComponent {
  title = 'Hello World';
}

To integrate the HelloComponent into the main application, you need to add the component import and declaration in the app.component.ts file (or app.module.ts in case of NgModule) and use the <app-hello> selector in the app.component.html template.

<app-hello></app-hello>

Run the following commands from the project directory:

npm install
It installs the dependencies listed in the package.json file into the local node_modules directory (for initial setup)

npm start
or 
ng serve
It builds and runs the Angular project and starts the embedded development server. Open your browser on http://localhost:4200/

If necessary, change the default port number: 
npm start -- --port 8081

Notes:
  • A service class can be created to handle the business logic.
  • Polyfills, ESLint, and end-to-end (E2E) testing can optionally be used to enhance browser compatibility, code quality, and comprehensive application testing.
  • In Angular v14 and above, standalone components make it easier to create components without NgModule. In Angular v19, standalone: true is the default setting for components, directives, and pipes.
  • In Angular v13 (NgModule-based), you import the FormsModule in the @NgModule of mycomponent.module.ts (for larger apps) or app.module.ts (for smaller apps). In Angular v19, you import the FormsModule directly in the standalone component that requires form functionalities.
