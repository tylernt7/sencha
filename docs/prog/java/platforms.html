Java Platforms

Platforms

  • Java SE: Java Platform, Standard Edition
    • Core Java
  • Java EE: Java Platform, Enterprise Edition
    • Jakarta EE 8 - 10 (2019 - 2022)
    • Java EE 5 - 8 (2006 - 2017)
    • J2EE 1.2 - 1.4 (1999 - 2003)
    • Application servers: Glassfish (Grizzly Web Server), Tomcat (Catalina Web Server), WildFly/JBoss (Undertow Web Server), Oracle WebLogic (built-in Web Server)
    • Web servers: Nginx (pronounced "engine x"), Apache, Jetty, Mongoose, IIS (Internet Information Services)
    • Frameworks: Spring, Hibernate, Struts, JSF, GWT
  • Java ME: Java Platform, Micro Edition
    • Java for Mobile Devices
  • Java FX: Java Platform, Java FX ScriptTM
    • Since JDK 9, Java FX is provided as a separate package, but removed in JDK 11
    • FXML: XML-based user interface markup language

Difference between web server and application server

A web server is a subset of an application server. A web server delivers static web content - HTML pages, files, images, video — in response to HTTP requests from a web browser. An application server delivers static and dynamic content, such as transaction results, decision support, or real-time analytics. A web server includes reverse proxy (gateway between the outer firewall and the inner firewall) and load balancing (automated traffic distribution using e.g. round-robin algorithm). By placing a web server in front of an application server, performance and security can be ensured (static content, load balancing).

How to manage Nginx on Linux?

sudo service nginx start
sudo service nginx stop
sudo service nginx restart
sudo service nginx reload
OR 
sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx
sudo systemctl reload nginx
Note:
  • To run commands with superuser privileges as the system administrator (root), use the sudo command
  • The systemctl is a more powerful version of service. With systemctl you can use more advanced commands
